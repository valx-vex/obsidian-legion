## Source Article Structure

Keep it tight -- 50-100 words per source card:

- **What This Source Is**: One sentence identifying the document type and subject.
- **Key Takeaways**: 2-4 bullet points capturing the most important information in the source. What would someone need to know before deciding to read the full document?
- **Related Articles**: Link to existing VEXPEDIA articles that this source feeds into using [[wikilinks]]. Every source connects to something.

## Voice & Style

- Concise but not lifeless. Even a stub can have personality.
- Think of source cards as trail markers: they tell the reader what is ahead and where it connects.
- Use [[wikilinks]] to link the source to every relevant entity, concept, and event in the wiki.
- Always reference the raw file path so the reader can find the original.
